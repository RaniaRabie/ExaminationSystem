## Project Approach (Step by Step)

We didn’t jump directly into coding — instead, we followed a clear database lifecycle:

### 1️⃣ EERD Design

* Designed the full system using Enhanced Entity Relationship Diagram
* Defined entities, relationships, and constraints

### 2️⃣ Database Mapping

* Converted EERD into relational schema
* Defined primary keys, foreign keys, and table structures

### 3️⃣ Database Implementation

* Created tables using SQL Server
* Applied constraints (PK, FK, CHECK, NOT NULL)
* Organized database into schemas

### 4️⃣ Data Insertion

* Inserted realistic sample data
* Ensured data consistency and integrity

### 5️⃣ Security & Administration

* Implemented roles and permissions
* Managed login access
* Created database backup scripts
